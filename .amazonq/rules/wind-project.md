# Wind プロジェクトルール

## リリース時チェックリスト

コードを変更してGitHubにプッシュする前に、以下を必ず確認・更新すること：

1. **バージョン番号** (`YYYYMMDD.N` 形式) を3箇所すべて更新
   - `docs/index.html` — `<div id="version">` の表示テキスト
   - `docs/sw.js` — `CACHE_NAME` の値
   - `README.md` — ピン留めREADMEの内容と乖離がないか
2. **README.md** — 変更した機能の説明が実態と合っているか確認

## コード変更ルール

- ソースコードを変更する前に、変更内容と理由を説明して許可を得ること
- 許可なくコードを編集しない
